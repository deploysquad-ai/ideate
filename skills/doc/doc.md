# Doc — Generate Session Document

This skill assembles all extracted artifacts and merged conclusions into a structured Markdown document — the primary deliverable of every ideation session.

## Usage

- `/ideate.doc` — generate (or regenerate) the session document

## Generation Process

1. **Read session metadata.** Read `.ideate/session.md` to get the project name.

2. **Read all artifacts.** Read every file in `.ideate/artifacts/`. Group them by type:
   - Goals
   - Personas
   - Constraints
   - Modules
   - Decisions
   - Features

3. **Read the main log.** Read `.ideate/main.md` for the sequence of merged conclusions.

4. **Assemble the document.** Write to `.ideate/output/<project-name-slugified>.md` using this template:

```
# <Project Name>

> Generated by Ideate on <today's date>. <N> artifacts across <M> merged branches.

## Overview

<Synthesize a 2-3 paragraph overview from the main log. This is NOT a copy of main.md — it's a narrative summary of what was explored and decided during the session.>

## Goals

<For each Goal artifact, include its name and description. If no goals were extracted, omit this section.>

### <Goal Name>
<Description from the goal artifact>

**Success Criteria:** <from the goal artifact>

## Personas

<For each Persona artifact>

### <Persona Name>
<Description>

**Goals:** <from the persona artifact>
**Context:** <from the persona artifact>

## Constraints

<For each Constraint artifact>

### <Constraint Name>
<Description>

**Scope:** <from the constraint artifact>
**Implications:** <from the constraint artifact>

## Modules

<For each Module artifact>

### <Module Name>
<Description>

**Responsibilities:**
<from the module artifact>

**Boundaries:**
<from the module artifact>

## Decisions

<For each Decision artifact>

### <Decision Name>
**Context:** <from the decision artifact>
**Decision:** <from the decision artifact>
**Rationale:** <from the decision artifact>
**Alternatives Considered:** <from the decision artifact>

## Features

<For each Feature artifact>

### <Feature Name>
<Description>

**Rationale:** <from the feature artifact>
**Open Questions:** <from the feature artifact, if any>

---

## Session Log

<Include the full contents of main.md — the sequence of merged conclusions. This provides the chronological narrative of how the document was built.>

---

## Unconfirmed Items

<List any artifacts with Status: draft that were never confirmed. Flag them:>

> The following items were extracted but not explicitly confirmed during the session:
> - <Type> - <Name>
> - <Type> - <Name>
```

5. **Present to the user.** After writing the file, say:
   > "Document generated at `.ideate/output/<filename>.md`.
   > **Summary:** <N> artifacts (<confirmed count> confirmed, <draft count> still draft).
   > <List any sections that are empty or thin — e.g., 'No personas were extracted — you may want to revisit who this is for.'>
   > Want me to read it back to you, or make changes to any artifacts before regenerating?"

6. **Update session status.** Update `.ideate/session.md` to set `Status: complete`.

## Regeneration

If the user runs `/ideate.doc` again after making changes (updating artifacts, merging more branches), overwrite the existing output file with the latest state. Say: "Document regenerated with latest changes."

## Gap Detection

Before generating, scan for common gaps and warn the user:
- No personas? "No personas extracted — the document won't have a 'who is this for' section."
- No constraints? "No constraints captured — consider if there are technical or business limits worth documenting."
- All artifacts still draft? "All artifacts are still in draft — none were explicitly confirmed. The document will flag these."
- No features? "No features extracted — the document will be light on 'what does it do'."

Let the user decide whether to address gaps or generate anyway.
